# Engineering Backlog

Этот файл хранит полезные, но не обязательные для ближайшей упаковки MVP инженерные шаги.

## Отложенный план

1. `LLM cache edge cases`
   - проверить битые `llm_summary`;
   - обработать устаревшую схему ответа;
   - не использовать старый summary, если payload уже изменился;
   - аккуратно разрулить fallback по дате, если он больше не подходит.

2. `Browser bridge / URL flow`
   - проверить поведение без расширения;
   - проверить позднее подключение расширения;
   - убедиться, что старый URL не залипает;
   - отфильтровать мусорные URL и пустые состояния.

3. `Idle / tracker consistency`
   - проверить быстрые переходы `idle -> active`;
   - проверить, что не появляются дубли или дырки в интервалах;
   - убедиться, что простои и активные записи не конфликтуют.

4. `Stoplist end-to-end`
   - прогнать сценарий `app`;
   - прогнать сценарий `site`;
   - убедиться, что UI, tracker и browser bridge работают согласованно.

5. `Export integrity`
   - проверить полноту JSON-экспорта;
   - проверить даты, пустые блоки и дубли;
   - убедиться, что экспорт пригоден для бэкапа и ручной диагностики.

6. `Final functional pass`
   - пройти все ключевые пользовательские сценарии;
   - собрать короткий список реальных остаточных багов;
   - решить, что из них идет в релиз, а что откладывается.
