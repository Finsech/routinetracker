# Fast-Track Plan

Цель этого плана — за минимальное число шагов дойти до упакованного рабочего приложения, не распыляясь на второстепенные инженерные улучшения.

## Короткий путь до готового приложения

1. `Заморозить scope MVP`
   - не брать новые функции;
   - чинить только то, что мешает запуску, трекингу, группировке и базовому использованию.

2. `Прогнать один полный сценарий пользователя end-to-end`
   - запуск трекера;
   - запись активностей;
   - фиксация простоя;
   - LLM-группировка;
   - просмотр `Сегодня`, `Недели`, `Аналитики`;
   - экспорт JSON.

3. `Исправить только блокирующие баги`
   - падения;
   - зависания;
   - пустые экраны;
   - явно сломанный трекинг;
   - сломанная LLM-группировка;
   - критически битый экспорт.

4. `Собрать release-ready пакет`
   - проверить `npm run build`;
   - проверить `cargo check`;
   - собрать desktop build;
   - убедиться, что приложение стартует как обычный пользовательский бинарник.

5. `Подготовить релизный комплект`
   - кратко зафиксировать, что готово;
   - перечислить known limitations;
   - подготовить changelog / release notes;
   - затем перейти к упаковке и приватной публикации.
