# Roadmap FocusFlow

## Статус

Проект находится на этапе ранней реализации MVP.

Выполнено:

- установлен локальный toolchain для Tauri: Rust, Cargo, MSVC Build Tools, WebView2;
- создан официальный scaffold Tauri 2 + React + TypeScript;
- подключены Tailwind CSS и shadcn/ui;
- обновлены зависимости Vite, audit чистый;
- создан первый русский UI-прототип на мок-данных;
- frontend разложен на страницы, app-shell, dashboard-компоненты и отдельные мок-данные;
- добавлен SQLite-слой на Rust-стороне: миграции, activity log, settings, stoplist;
- добавлен frontend API-клиент с fallback на мок-данные для браузерного preview;
- экран настроек подключен к Tauri API и SQLite, при browser preview остается на fallback-данных;
- добавлен базовый Rust-модуль трекинга с Tauri-командами `start_tracking`, `stop_tracking`, `get_tracking_status`;
- первый слой трекинга в desktop runtime использует Windows foreground-event hook для быстрой реакции на смену активного окна, сохраняет имя процесса, заголовок окна и интервалы активности в SQLite; polling остается резервным слоем;
- добавлен базовый учет простоя: после 10 минут без ввода мыши или клавиатуры текущая сессия закрывается, idle-интервал пишется в SQLite, новые активные интервалы не пишутся до возвращения активности;
- в боковой панели UI добавлена кнопка запуска и остановки трекинга, live-статус текущей активности и счетчик времени без ввода с мок-режимом для browser preview;
- шапка приложения показывает текущую дату через локальное форматирование, без хардкода демонстрационной даты;
- главный экран и история подключены к `activity_log` и `idle_log`, регулярно обновляют данные из API и показывают реальные активные и idle-интервалы вместо статичных моков;
- добавлен первый review-flow простоя: неуточненный idle-интервал открывает закрываемую модалку, где можно сохранить заметку, игнорировать период или отложить уточнение;
- добавлена защита от коротких шумовых интервалов: активные сессии короче 5 секунд не сохраняются в SQLite;
- добавлен Windows foreground-event hook: смена активного окна обрабатывается через системное событие, а polling раз в 2 секунды остается страховкой;
- добавлен минимальный browser bridge: Tauri слушает `127.0.0.1:17653`, расширение из `browser-extension/` отправляет URL и заголовок активной вкладки, а трекер сохраняет свежий URL для браузерных процессов;
- экран настроек получил управление stoplist-приложениями; трекер пропускает процессы из stoplist при записи активности;
- при закрытии окна Tauri трекер явно останавливается и закрывает текущую сессию; добавлена Drop-страховка фонового потока;
- добавлен frontend-слой агрегации активностей: дневной таймлайн, активное время, недельная история, heatmap-уровни и временная группировка по приложениям;
- добавлен frontend-слой подготовки LLM payload и карточка `LLM-группировка` на главном экране с копированием JSON;
- подключен первый локальный LLM-провайдер Ollama: настройки адреса и модели вынесены в экран настроек, главный экран умеет вручную отправлять дневной payload в Ollama и отображать результат как потоки и стримы;
- установлена и проверена рабочая локальная модель `qwen2.5:7b-instruct`; дефолт `gpt-oss:20b` заменен, потому что локальный blob модели не загружался в Ollama;
- LLM-группировки сохраняются в SQLite в таблицу `llm_summary` и автоматически переиспользуются для того же дневного payload, провайдера и модели без повторного запроса к Ollama;
- добавлен просмотр состава стрима: клик по стриму на главном экране открывает модальное окно с исходными активностями, приложениями, временем и длительностью;
- добавлен экспорт данных в JSON из экрана настроек: активные интервалы, простои, настройки, stoplist и сохраненные LLM-группировки;
- desktop dev mode запущен через Tauri; подтверждена запись реальных интервалов в SQLite (`comet.exe` и заголовки активных окон);
- проверены `npm run build` и `cargo check`.

Принятые решения:

- основной проект разрабатывается без оркестратора;
- идея оркестратора сохранена отдельно в `orchestrator/`;
- все, что описано в `SPECIFICATION.md`, считается объемом MVP до появления новых ограничений;
- стартовый язык интерфейса - русский;
- промежуточные UI-варианты желательно смотреть через браузерный dev server на мок-данных;
- полноценная проверка native-функций выполняется через Tauri desktop dev mode;
- будущий репозиторий GitHub должен быть приватным;
- приватная публикация в GitHub откладывается до завершения визуального редизайна;
- roadmap редизайна зафиксирован в `REDESIGN_ROADMAP.md`.

3. Подготовить приватную публикацию в GitHub после визуального редизайна.
