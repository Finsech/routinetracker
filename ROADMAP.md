# FocusFlow Roadmap

## Текущий статус

- Текущий публичный релиз: `0.1.4`
- MVP собран, опубликован и обкатывается на реальных данных.
- Основной рабочий контур: **Windows desktop**
- macOS: есть unsigned build через GitHub Actions, подходит для проверки и open-source дистрибуции

## Что уже готово

### Трекинг и локальная база

- автоматический трекинг активных окон;
- учет простоев и подтверждение idle-интервалов;
- локальное хранение в SQLite;
- работа в системном трее;
- single-instance режим;
- автозапуск для Windows;
- browser URL bridge для уточнения браузерной активности.

### Экраны приложения

- `Сегодня`
- `Анализ дня`
- `Неделя`
- `Настройки`

### LLM-группировка

- локальная интеграция с Ollama;
- ручная сборка дня;
- автосаммари по таймеру;
- `chunk + merge` для длинных рабочих дней;
- hybrid preclassification перед LLM;
- словарь коммуникационных приложений и web-версий;
- `project_hint` и `project_key` для лучшего определения проектов внутри `Работы`.

### Аналитика и экспорт

- day/week таймлайны;
- годовая heatmap активности;
- блоки `Твой день`, `Картина дня`, `Потоки дня`;
- экспорт `JSON` и `CSV`.

## Что обкатываем сейчас

### Приоритет 1 — стабильность LLM

- длинные реальные дни;
- корректность потоков `Работа / Обучение / Общение / Прочее`;
- устойчивость `chunk + merge`;
- точность определения project aliases внутри `Работы`.

### Приоритет 2 — фоновые сценарии

- автосаммари каждые 30 минут;
- сброс таймера после ручного `Обновить группы`;
- `tray -> вернуть окно`;
- `sleep / wake`;
- сохранение состояния после возврата из фона.

### Приоритет 3 — post-release багфиксы

- точечные проблемы UI;
- редкие кейсы с кодировкой;
- проверка macOS runtime-пути на реальных устройствах.

## Рефакторинг

Отдельная спецификация:

- `REFACTORING_SPEC.md`

Статус:

- Пакет A — завершен
- Пакет B — завершен
- Пакет C — завершен

Что уже сделано:

- введены стабильные `FlowId` и `SettingKey`;
- базовые русские строки вынесены в единый слой;
- критичная логика отвязана от сравнений по русским подписям;
- дефолты frontend/backend приведены к общему контракту;
- добавлен guardrail против mojibake;
- LLM prompt strings, frontend error copy и backend service helpers начали жить раздельно, а не вперемешку.

Что осталось по пакету C:

- пакет закрыт;
- дальше возможны только точечные доборы по мере новых фич, без отдельного санитарного долга.

## Следующий релизный цикл

Цель следующего цикла:

1. добрать фидбэк по `0.1.4`;
2. закрыть реальные баги без разрастания scope;
3. выпустить стабилизационный релиз после накопления достаточного набора правок.
