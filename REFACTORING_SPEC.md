# FocusFlow Refactoring Spec

## Цель

Убрать хрупкие зависимости логики от пользовательских русских строк, собрать UI-copy в один слой и подготовить кодовую базу к дальнейшей чистке без точечных аварийных правок.

## Принципы

1. Сначала чиним слои, которые реально создают баги.
2. Не меняем продуктовый смысл экранов и потоков, если это не требуется рефакторингом.
3. Логику переводим на стабильные идентификаторы, а UI оставляем на человекочитаемых русских подписях.
4. Документацию обновляем после каждого завершенного пакета.

## Порядок работ

### Пакет A — критичный

1. Ввести стабильные идентификаторы потоков и настроек.
2. Вынести базовые русские строки в единый слой `src/lib/copy/ru.ts`.
3. Отвязать `Сегодня` и `Настройки` от сравнений по русским подписям.
4. Перевести вычисления `Фокуса` и критичные UI-ветки на `flow.id` и `setting.key`.

### Пакет B — стабилизация

1. Убрать дубли дефолтов между frontend mock/default слоем и backend seed-слоем.
2. Привести mock-данные и реальные типы к одному контракту.
3. Свести разрозненные flow/setting label mapping в единый источник правды.

### Пакет C — санитария

1. Вычистить старые разделы `ROADMAP.md` и `SPECIFICATION.md` от битой кодировки.
2. Добавить guardrail против mojibake и случайного возврата кривых строк.
3. Разнести UI-copy, backend errors и LLM prompt strings по разным слоям.

## Текущий проход

- **Пакет A**: завершен.
- **Пакет B**: завершен.
- **Пакет C**: завершен.

## Definition of Done для Пакета A

1. В коде есть стабильные `FlowId` и `SettingKey`.
2. `TodayPage` не сравнивает потоки по строкам `"Работа"` и `"Обучение"`.
3. `SettingsPage` не управляет состоянием через `row.label === "Автозапуск"`.
4. Базовые русские подписи навигации, потоков и настроек живут в одном месте.
5. `npm run build` и `cargo check` проходят.

## Статус выполнения

### Сделано в Пакете A

1. Добавлены стабильные `FlowId` и `SettingKey`.
2. Базовые русские подписи вынесены в `src/lib/copy/ru.ts`.
3. `TodayPage`, `SettingsPage`, `AnalyticsPage` и часть dashboard-компонентов отвязаны от критичных сравнений по русским строкам.

6. LLM prompt strings вынесены в отдельный слой `src/lib/llm/prompts.ts`.
